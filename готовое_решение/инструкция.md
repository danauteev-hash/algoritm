# Инструкция

В этой папке лежит алгоритм распределения учеников по трекам.

## Что внутри

- `распределение_по_трекам.py` — рабочий скрипт
- `образец_входных_данных.xlsx` — пример входного файла с обязательными и дополнительными колонками
- `выход\\результат_распределения.xlsx` — итоговый файл с несколькими листами:
  - `Результат` — назначения и отметка о выборе трека
  - `Гипотеза` — краткая сводка по режиму распределения и итоговый вывод
  - `Количество_по_трекам` — число студентов по каждому треку
  - `Без_выбора` — отдельный список студентов без выбранного трека
  - `Приоритеты_1_2` — списки студентов, у которых трек стоит в приоритетах 1 или 2

В режиме `только предпочтения` колонки рейтинга на листе `Результат` не используются и остаются пустыми.

## Как запускать

1. Откройте `распределение_по_трекам.py`.
2. В самом низу файла измените путь прямо в вызове `main(...)`.
3. Должно получиться примерно так:

```python
if __name__ == "__main__":
    main(r"C:\Users\A006\Downloads\Рейтинг_40.xlsx")
```
4. Запустите:

```powershell
py C:\Users\A006\Desktop\пипе\готовое_решение\распределение_по_трекам.py
```

## Что требуется во входном файле

Скрипт ожидает `xlsx`, по структуре `Рейтинг.xlsx` или `Рейтинг_40.xlsx`:

- строка над заголовками может содержать подписи семестров: `1 семестр`, `2 семестр`, `3 семестр`
- строка заголовков должна содержать:
  - `id`
  - оценки по предметам
  - колонки треков: например `Трек А`, `Трек В`, `Трек С`

## Какие поля обязательны

- `id`
- оценки по предметам
- приоритеты треков

## Какие поля дополнительные

Если есть, они будут учтены автоматически:

- `Посещаемость %`
- `Внеучебные мероприятия`
- `Олимпиады`
- `Конкурсы`
- `Статус без выбора`

Если их нет, скрипт продолжает работать без ошибки.

## Студенты без выбора трека

Если у студента по всем трекам стоит `0`, он помечается как студент без выбора трека.

Дополнительно можно указать колонку:

- `Статус без выбора`

Примеры значений:

- `сделает выбор позднее`
- `уйдет`

Эти студенты будут:

- отмечены на листе `Результат`
- вынесены отдельно на лист `Без_выбора`

## Олимпиады с разными весами

Для олимпиад теперь поддерживаются два варианта:

1. Простой:
   - `Олимпиады`

2. Подробный:
   - `Олимпиада 1`
   - `Вес олимпиады 1`
   - `Олимпиада 2`
   - `Вес олимпиады 2`
   - и так далее

В подробном варианте скрипт считает вклад олимпиад как сумму:

`значение олимпиады * ее вес`

по всем найденным парам колонок.

Пример:

- `Олимпиада 1 = 1`, `Вес олимпиады 1 = 2.5`
- `Олимпиада 2 = 1`, `Вес олимпиады 2 = 1.0`

Тогда итоговый олимпиадный вклад студента до нормализации будет равен `3.5`.

Если в файле есть подробные колонки по олимпиадам, именно они считаются основными. Если их нет, используется простая сводная колонка `Олимпиады`.

## Логика рейтинга

Этот блок нужен для режима, где распределение строится с учетом оценок.
Но по умолчанию скрипт сейчас запускается в режиме:

- `только предпочтения`

Это значит:

- оценки читаются из файла, но не участвуют в самом распределении;
- распределение строится только по трекам и их приоритетам;
- цель — получить максимально ровную загрузку треков и при этом как можно лучше учесть выбор студентов.

1. Чем более поздний семестр, тем больше его вес.
2. Учитывается средняя оценка по семестрам.
3. Работает механизм подтвержденного прогресса:
   - выше, чем в прошлом семестре — рейтинг растет
   - та же оценка — без изменения
   - ниже — рейтинг уменьшается
4. Дополнительные показатели усиливают рейтинг по своим коэффициентам.
5. Для олимпиад можно задавать индивидуальные веса по отдельным колонкам.
6. Добавлен малый антисовпадающий компонент, чтобы одинаковые значения рейтинга встречались как можно реже.

## Логика треков

- `1` — самый желаемый трек
- `2` — второй по желательности
- `3` — третий по желательности
- `0` — безразлично, можно назначать в этот трек без потери приоритета

В режиме `только предпочтения` скрипт ищет распределение, которое:

1. соблюдает равномерную загрузку треков;
2. максимально сохраняет первые приоритеты;
3. затем старается сохранить вторые приоритеты;
4. отдельно помечает студентов без выбора трека.

## Что появляется в результате дополнительно

1. Количество студентов по трекам.
2. Отдельная пометка студентов без выбора трека.
3. Список студентов по каждому треку, если этот трек стоит у них в приоритете `1` или `2`.
